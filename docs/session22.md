## Notes

- The compile milestone is done, but most of the new bridge members are
  honest stubs (no-ops or degenerate returns). The control root compiles and
  the WPF logic runs, but rows/cells are not generated, columns have no
  width computation, and details/virtualization are inert. Behavior comes
  from replacing stubs rung by rung, driven by a runtime sample.
- A probe artifact (`DataGrid.cs.bak`) was committed with the session work
  and removed afterwards.
- Build-flag gotcha discovered while probing: `-p:DefineConstants="HAS_UNO"`
  on the CLI *replaces* the csproj's constant list (dropping `WINUI_BRIDGE`
  and friends) and produces hundreds of misleading errors. Build with the
  project defaults; `HAS_UNO` is already defined there.
